Задание:

Реализовать api для системы учёта клиентов автостоянки. Система должна иметь функции создания, редактирования, удаления данных о клиентах и их автомобилях. Должна присутствовать методы ведения учёта того, сколько и какие автомобили находится на стоянке.

При написании проекта следует обратить внимание на защиту от XSS атак и SQL–инъекций, а также на то, чтобы архитектура соответствовала MVC принципу.

Основные сущности:
Клиент
Автомобиль

Атрибуты сущности "Клиент":
ФИО (мин 3 символа)
Пол (обязательный)
Телефон (обязательный, уникальный)
Адрес
Автомобил(ь/и) (обязательный, мин 1 шт)

Атрибуты сущности "Автомобиль":
Марка (обязательный)
Модель (обязательный)
Цвет кузова (обязательный)
Гос Номер РФ (обязательный, уникальный)
Флаг статуса автомобиля означающий, что авто находится или отсутствует на стоянке (обязательный)

Требования к используемым технологиям:
PHP 7.4
MySQL 8 и выше или MariaDB 10.3 и выше
Обязательно использовать фреймворк Laravel последних стабильных версий. Все запросы к БД должны быть написаны через QueryBuilder или Raw Query (т. е. без методов Eloquent ORM).
Документацию по api необходимо написать в программе postman (файл экспорта коллекции v2.1 добавить в репозирий)
